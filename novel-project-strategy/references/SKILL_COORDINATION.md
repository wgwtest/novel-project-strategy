# Skill Coordination

Use this reference when `novel-project-strategy` overlaps with other skills.

## Precedence

Apply rules in this order:

1. explicit user instruction
2. project-local workspace rules
3. `novel-project-strategy`
4. defaults from other skills

## Coordination boundaries

### With `novel-writing`

- Use `novel-project-strategy` for workspace governance, state files, recovery order, structural sync rules, and manuscript-layer discipline.
- Use `novel-writing` for chapter planning, prose drafting, revision judgment, scene structure, character introduction quality, and style fidelity.
- Do not let project-governance rules overwrite prose-craft judgments, and do not let prose-craft workflows replace chapter-state or recovery discipline.

### With `project-engineering-strategy`

- Use `project-engineering-strategy` for code repositories, toolchains, worktrees, execution contracts, and engineering acceptance governance.
- Use `novel-project-strategy` for longform fiction workspaces where chapters, outline, setting, and timeline are the primary work objects.
- In mixed repos that contain both code and fiction workflows, follow the project-local mapping for which surface owns which files.

## Non-goals

- Do not use this skill to replace project-specific worldbuilding law.
- Do not use this skill to replace direct prose-craft review.
- Do not merge all novel-related workflows into one oversized skill when narrower boundaries remain clear.
