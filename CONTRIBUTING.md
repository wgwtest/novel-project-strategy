# Contributing

Thanks for improving this skill.

## What Helps Most

- stronger recovery and reload-order examples
- clearer chapter-state, confirmation-state, and sync-state rules
- better boundaries between project governance and prose craft
- public-safe reference material that does not depend on private manuscripts
- packaging and installation improvements that reduce friction

## Contribution Rules

- Keep the installable package inside `novel-project-strategy/`.
- Keep root files human-facing and package files skill-facing.
- Do not add private manuscript or local workspace residue.
- Prefer concrete workflow models over generic productivity advice.
- If you change workflow rules, update examples and root documentation in the same change.

## Pull Request Checklist

- explain the workflow problem or ambiguity being fixed
- keep the scope narrow
- update docs if behavior or packaging changed
- verify the public package still installs cleanly from the `novel-project-strategy/` path

## Issues

Open an issue when reporting:

- install or upgrade problems
- trigger misfires
- unclear state or sync guidance
- weak recovery rules for longform projects
- ideas for high-value additions
